Database Models (in main.py):
==============================
1. User
   - id, username, email, password_hash
   - wallet_address (unique blockchain address)
   - token_balance, hours_earned, hours_spent
   - reputation_score, is_admin
   - created_at timestamp

2. Service
   - id, title, description, category
   - duration (hours), status, service_type
   - provider_id, requester_id
   - created_at, completed_at timestamps

3. Transaction
   - id, from_user_id, to_user_id
   - amount, transaction_type, description
   - service_id (linked service)
   - timestamp

4. Proposal
   - id, title, description
   - proposer_id, status
   - votes_for, votes_against
   - created_at, ends_at timestamps

5. Vote
   - id, proposal_id, voter_id
   - vote_type (for/against)
   - voting_power (based on tokens)
   - timestamp

Routes (URLs):
==============
Public Routes:
- /                           → Home page
- /how-it-works              → How it works guide
- /register                  → User registration
- /login                     → User login

Protected Routes (require login):
- /logout                    → User logout
- /dashboard                 → User dashboard
- /offer-service             → Offer service form
- /request-service           → Request service form
- /marketplace               → Browse service requests
- /accept-service/<id>       → Accept a service request
- /governance                → DAO governance page
- /create-proposal           → Create proposal form
- /vote/<id>/<type>          → Vote on proposal

Admin Routes (require admin privileges):
- /admin                     → Admin panel
- /admin/make-admin/<id>     → Grant admin privileges

Design Principles:
==================
1. Equal Value: 1 token = 1 hour (all services valued equally)
2. Trust-Based: Reputation system builds community trust
3. Decentralized: DAO governance for community decisions
4. Transparent: All transactions recorded and visible
5. Blockchain-Ready: Wallet addresses and transaction structure
